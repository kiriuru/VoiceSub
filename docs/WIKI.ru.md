## 0. Версия и обновления

Обновление:

1. Закройте программу.
2. Замените файл `Stream Subtitle Translator.exe`.
3. Запустите программу снова.

Если после обновления есть проблемы:

- используйте восстановление среды в стартовом окне;
- если не помогло — полный сброс среды;
- через командную строку можно использовать ключи восстановления и сброса.

---

## 1. Быстрый старт

1. Выберите профиль запуска.
2. В разделе `Распознавание` задайте:
   - `Способ распознавания`;
   - `Язык распознавания`;
   - микрофон.
3. Во вкладке перевода включите `Переводить распознанную речь`, если нужен перевод.
4. Нажмите кнопку запуска.
5. Проверьте текст в превью и в блоке `Переведённый результат`.

---

## 2. Что делать, если не работает

**Нет текста**

- не нажата кнопка запуска;
- не выбран микрофон;
- нет доступа к микрофону.

**Есть исходный текст, нет перевода**

- выключен чекбокс `Переводить распознанную речь`;
- нет включённых линий перевода;
- в блоке `Переведённый результат` отображается ошибка.

**Пусто в OBS**

- проверьте ссылку overlay;
- проверьте чекбоксы `Показывать оригинальный текст` и `Показывать переводы`;
- убедитесь, что распознавание запущено.

## 3. Профили запуска

Профили:

- быстрый запуск через браузерное распознавание;
- запуск с NVIDIA;
- запуск только на CPU;
- удалённый контроллер;
- удалённый воркер.

Важно:

- после быстрого веб-запуска локальный режим может быть временно недоступен до следующего запуска в локальном AI-профиле;
- в режиме worker браузерный путь распознавания не используется;
- для remote-сценария сначала запускают worker, потом controller.

---

## 4. Распознавание

Основные элементы:

- способ распознавания;
- язык распознавания;
- выбор браузера для отдельного окна распознавания в desktop-режиме;
- выбор локального AI-провайдера.

Пояснение:

- для браузерных режимов используется отдельное worker-окно;
- микрофон для браузерного режима выбирается в адресной строке окна worker.

---

## 5. Перевод

### 5.1 Основные чекбоксы

- Переводить распознанную речь;
- Использовать кэш переводов (пропускать дубликаты);
- Сохранять кэш переводов на диск между сессиями.

Как работает кеш:

- при включённом повторном использовании кэша одинаковая фраза не отправляется в провайдер снова;
- при включённом сохранении кэша на диск кэш остаётся после перезапуска программы.

### 5.2 Линии перевода

Раздел линий перевода:

- выбор целевого языка для новой линии;
- добавление и удаление линий;
- список линий перевода, где для каждой линии доступны:
  - включение/выключение;
  - целевой язык;
  - провайдер для этой линии;
  - метка линии (если поле отображается в текущей сборке).

Особенности:

- у каждой линии может быть свой провайдер;
- дубли целевых языков допустимы;
- максимально используется 5 линий перевода.

### 5.3 Настройки провайдера

Раздел настроек провайдера:

- провайдер по умолчанию для новых линий;
- выбор, настройки какого провайдера редактировать;
- поля в зависимости от провайдера:
  - API-ключ;
  - базовый адрес;
  - ссылка скрипта;
  - endpoint;
  - регион;
  - адрес провайдера;
  - модель;
  - загрузка рекомендуемых моделей;
  - выбор модели из списка;
  - показ всех моделей;
  - пользовательская инструкция для модели.

### 5.4 Переведённый результат

Раздел переведённого результата показывает:

- успешные переводы;
- ошибки (например, неверный ключ, недоступная модель, таймаут, ошибка сети).

## 6. Вывод субтитров

Раздел вывода субтитров.

### 6.1 Схема расположения строк в overlay

Поле схемы расположения строк:

- одна строка;
- две строки;
- стек.

Как это работает:

- режим одной строки — все видимые элементы выводятся одной строкой;
- режим двух строк — первый элемент отдельно, остальные объединяются во второй строке;
- режим стека — каждый элемент отдельной строкой.

### 6.2 Компактность

Чекбокс компактных отступов:

- уменьшает отступы между строками;
- полезно для маленькой зоны субтитров в OBS.

### 6.3 Видимость и лимит

- показывать оригинальный текст;
- показывать переводы;
- максимум строк перевода на экране.

Нюанс:

- включённых линий перевода может быть больше, чем показывается на экране;
- отображаются первые строки согласно текущему порядку.

### 6.4 Тайминги

Раздел таймингов субтитров:

- сколько держать завершённый исходный текст;
- сколько держать завершённый перевод;
- держать исходный текст, пока ещё виден его перевод;
- сразу заменять текущий блок при завершении следующей фразы.

### 6.5 Порядок строк

Кнопки порядка:

- выше;
- ниже.

Порядок влияет на:

- предпросмотр;
- overlay;
- режим первой видимой строки в OBS Closed Captions.

## 7. Стиль субтитров

Раздел стиля субтитров.

Базовый порядок настройки:

1. Выберите пресет.
2. Измените параметры стиля.
3. При необходимости используйте дополнительные параметры выбранного слота.
4. Нажмите кнопку сохранения пресета.
5. Сохраните config/профиль.

Важно:

- стиль общий для предпросмотра и OBS overlay;
- пользовательские пресеты можно удалять.

## 8. OBS Closed Captions

### 8.1 Подключение

Раздел OBS Closed Captions:

- отправлять субтитры в OBS Closed Captions;
- хост websocket;
- порт;
- пароль;
- что отправлять.

### 8.2 Что отправлять

Поле выбора отправляемого текста:

- выключено;
- исходный текст в живом режиме;
- только завершённый исходный текст;
- перевод 1 ... перевод 5;
- первая видимая строка.

### 8.3 Тайминги и дубликаты

- минимальный интервал между промежуточными обновлениями;
- минимальное изменение текста перед новой отправкой промежуточного обновления;
- задержка перед заменой текста финалом;
- через сколько миллисекунд очищать текст в OBS;
- не отправлять один и тот же текст повторно.

### 8.4 Зеркало для отладки

- зеркалить субтитры в обычный текстовый источник OBS для отладки;
- имя текстового источника для отладки;
- отправлять промежуточный текст в отладочный текстовый источник.

### 8.5 Twitch

Для зрителя:

- субтитры включаются кнопкой CC в плеере, если поток субтитров доступен.

Форматная совместимость:

- Twitch использует совместимый поток формата CEA-708/EIA-608;
- для стабильности лучше использовать простой текст;
- часть сложных символов может отображаться нестабильно в зависимости от плеера/платформы.

## 9. Характер распознавания

Раздел характера распознавания:

- как быстро появляется текст;
- как быстро речь считается законченной;
- насколько спокойно обновляется текст;
- RNNoise шумоподавление (экспериментально);
- сила RNNoise;
- пресет задержки Parakeet.

После изменений:

- сохраните config/профиль;
- перезапустите runtime: Стоп -> Старт.

## 10. Расширенные настройки распознавания

Раздел расширенных настроек распознавания:

- режим чувствительности к речи;
- пресет задержки;
- инкрементальное декодирование;
- режим выдачи промежуточного текста;
- минимум новых слов на промежуточный текст;
- частота обновления живого текста;
- сколько речи нужно до появления текста;
- сколько держать фразу открытой после короткой паузы;
- пауза, после которой фраза считается завершённой;
- максимальная длина фразы до разделения;
- минимальное изменение текста перед обновлением;
- склейка частых мелких обновлений;
- размер продвинутого аудио-окна;
- перекрытие между окнами;
- игнорировать слишком тихий вход до распознавания;
- игнорировать шум ниже порога;
- минимальная доля кадров, похожих на речь;
- сколько речи нужно до первого живого текста.

Рекомендация:

- ориентируйтесь на подсказки “по умолчанию / безопаснее” в UI;
- меняйте по одному параметру.

## 14. Web Speech

Страница worker содержит:

- возврат промежуточных результатов;
- непрерывный режим результатов;
- принудительное завершение промежуточного текста, если браузер не отдаёт финал;

- блок диагностики (статус, счётчики, websocket);
- поля живого промежуточного текста и последнего финального текста.

## 17. Глоссарий

- промежуточный текст — текст, который ещё может меняться;
- финальный текст — завершённая фраза;
- слот — отдельная линия перевода;
- overlay — страница вывода для OBS Browser Source;
- OBS Closed Captions — поток скрытых субтитров в OBS.
